<h1 id="what-is-shallow-copy">シャローコピーって何？</h1>

英語でshallow copy。意味は浅いコピー。

逆にdeep copy（ディープコピー）もあるみたい。

コピーに浅いも深いもあるってよくわかりませんね。コピーはコピーでしょと思うのですが、配列のコピーという記事でまとめたように

```jsx
const a = 1;
const b = a;
```

という変数の代入と

```jsx
const greetingList = [
  {id:0, title: "Hello."},
  {id:1, title: "Nice to meet you."},
  {id:2, title: "Good evening."},
  {id:3, title: "See you again!"}
];

const reverseGreetingList = [...greetingList];
```

オブジェクトの代入は違うことをしているようです↓。

<h2 id="try-it-out">実際にやってみよう</h2>

よく使うスプレッド構文。

```js
[...list]
```

これは1階層しかコピーできないのです。それより深い部分は参照となります。

こんなプログラムがあるとしましょう。

```jsx
const original = [
  { name: "A" },
  { name: "B" }
];
export default function Test() {
  const copy = [...original];
  copy[0].name = "Z";

  return (
    <>
      {original.map(o => (
        <p>oliginal : {o.name}</p>
      ))}
      {copy.map(c => (
        <p>copy : {c.name}</p>
      ))}      
    </>
  );
}
```

配列のコピーという記事でオブジェクトのコピーは参照の場所を渡しているというふうに書きました。となるとoriginal[0].nameとcopy[0].nameは同じ場所を参照しているので、copyを直接変えるとoriginalも変わるということになります。

結果
```md
oliginal : Z
oliginal : B
copy : Z
copy : B
```

確かにcopyしか変えていないのにoriginalもZになりましたね。

そう、これがシャローコピー。

スプレッド構文は「新しい配列」を作りますが、中の要素（オブジェクトや配列）は同じ参照になります。

わかりやすく中身をobjAとobjBという形にしてみましょう。

[...original]で展開されているのはオブジェクトの参照になります。

```js
copy = [...original]
```

これは

```js
original = [ objA, objB ]
copy = [ objA, objB ]
```

ということ。

それをrevese()メソッドで逆順にしてみましょう。

```js
const copy = [...original];
copy.reverse();
```

これをobjAとobjBで表すと

```js
original = [ objA, objB ]
copy = [ objB, objA ]
```

と参照の位置を入れ替えただけになります。

ではオブジェクトの中身を書き換える記述ではどうなるのでしょうか。

```js
copy[0].name = "Z";
```

この記述は

```js
objA.name = "Z"
```

をしていることになります。
objAはoriginalも参照しているので

```js
original[0].name = copy[0].name = objA.name = "Z"

// もう少し具体的に
original = [ objA(Z), objB ]
copy = [ objA(Z), objB ]
```

こんな形で中身が入れ替わってしまうのです。

共通のobjAの中身を書き換えてしまったがためにこのようなことが起こってしまうのです。

この章の冒頭で「二つのプログラムの違いはオブジェクトを直接書き換えたかそうでないかです。」と書きましたが、破壊的メソッドは参照を入れ替えたり追加したりするだけで中身の変更をしていません。だから違う結果になりますが、参照元を書き換えてしまうとそれを元にしてしまうので同じ結果になってしまうのです。

<h3 id="destructive-methods-note">3/19追記 破壊的メソッドの参照について</h3>
破壊的メソッドは参照を入れ替えたり追加したりするだけで中身の変更をしていません。

と記載しましたが正しくは

破壊的メソッドは配列そのもの（並びや要素）を変更しますが、中に入っているオブジェクトの中身までは変更しません。


まるで伝言ゲームのようですね。

備考で書いた

```js
copy[0] = {name: "Z"};
```

これがちゃんと反映されるのはcopy[0]にobjCという別の参照を渡すことでoriginalには影響が出ないようになっていたのです。

```js
copy = [ objC(元objAだった場所), objB ]
```

<h2 id="what-shallow-copy-means">結局シャローコピーとはなんなのか？</h2>

一旦スプレッド構文に戻ってみましょう。

オブジェクトのない普通の配列をそのまま表示するプログラムを書いてました。

```jsx
const original = [1, 2, 3];
export default function Test() {
  const copy = [...original];
  copy[0] = "ZZZ";

  return (
    <>
      {original.map(o => (
        <p>oliginal : {o}</p>
      ))}
      {copy.map(c => (
        <p>copy : {c}</p>
      ))}      
    </>
  );
}
```

結果はこちら

```md
oliginal : 1
oliginal : 2
oliginal : 3
copy : ZZZ
copy : 2
copy : 3
```

当然こうなります。

書き換えたcopy[0]だけがZZZとなりoriginal[0]は1のままです。

次に2次元配列のプログラムを書いてみました。（本来はkeyなどを設けた方が良いのですが、省略します）

```jsx
const original = [[1,2,3], [4,5,6]];
export default function Test() {
  const copy = [...original];
  copy[0][1] = "ZZZ";

  return (
    <>
      {original.map(o => (
        <>
        {o.map(i => (
          <p>oliginal : {i}</p> 
        ))}
        </>
      ))}
      {copy.map(o => (
        <>
        {o.map(i => (
          <p>copy : {i}</p> 
        ))}
        </>
      ))}
    </>
  );
}
```

結果はこうなりました。

```md
oliginal : 1
oliginal : ZZZ
oliginal : 3
oliginal : 4
oliginal : 5
oliginal : 6
copy : 1
copy : ZZZ
copy : 3
copy : 4
copy : 5
copy : 6
```

copyの[0][1]番目（元は2）の部分をZZZに書き換えるとoriginalもZZZになりました。

最初に「スプレッド構文は1階層しかコピーしない」と書きました。オブジェクトのコピーと同じように中身が共有されているんですね。

実際はこんな感じでコピーされています。

```js
original = [ arrA, arrB, arrC ]
copy = [ arrA, arrB, arrC ]
```

オブジェクトの場合もこんな感じでしたね。

```js
original = [ objA, objB ]
copy = [ objA, objB ]
```

もっと中身もコピーしたい場合、ディープコピーを使います。

```js
const copy = original.map(arr => [...arr]);
copy[0][1] = "ZZZ";
```

こうすることで外も中も新しい配列になり元の配列には影響が出ません。

<h3 id="reverse-note">追記3/19 元の配列とrevese()メソッド</h3>

revese()メソッドで元の配列を弄ったのに元の配列はちゃんと表示されています。

このように記述しましたが

reverse()は呼び出した配列を変更する破壊的メソッドです。

ただし今回のコードでは

```js
const reverseGreetingList = [...greetingList].reverse();
```

と「コピーした配列」に対してreverse()を使っているため、
元のgreetingListは変更されません。

コピーを逆順にしているだけでした。

<h2 id="shallow-copy-is-tricky">シャローコピーって面倒だな</h2>

先ほどリンクを載せたTEST KINGのConsiderations for Shallow Copying（浅いコピーに関する考慮事項）の欄でこのようなことが書かれています。

「浅いコピー方法は、一般的に深いコピーよりも高速で、必要なメモリも少ないため、パフォーマンスが優先され、データ構造が単純なシナリオに最適です。」

私は勘違いをしていました。

シャローコピーは不完全なコピーでいちいち中身を参照ではなく値をコピーする手間をかける必要があるものだと思っていました。

本当は毎回ディープコピー（完全コピー）をするとパフォーマンスの低下につながります。

手書きでお知らせを人数分書いて配布するより一枚だけ書いて掲示板に貼ってみてもらった方が楽ですよね？

そんな感じでシャローコピーはプログラマーが軽くて速いコピー手段として選択できる方法の一つなのです。

中身を変えたければディープコピーを、順番を変える程度ならシャローコピーを、そんな感じで使い分けましょう。
